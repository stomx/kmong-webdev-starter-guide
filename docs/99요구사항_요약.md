# 요구사항 정리본

## 1) 현재까지 상황 요약
- 사용자는 현재 제공된 템플릿/페이지 구성에 대해 만족도가 낮아, **브랜딩 및 전반적 품질 개선**을 반복 요청함.
- 사용자 표현이 강한 언어이므로, 향후 응답은 예민한 반응을 피하고 **요청 반영 위주의 결과 제시**로 진행해야 함.
- 사용자는 ‘일단 결과 보고/반영 후 판단’ 흐름을 선호함.

## 2) 본 사용자의 핵심 요구사항 (즉시 반영 우선순위)
1. **템플릿 다채로움 강화**
   - Template A/B/C가 단순 샘플형으로 보이지 않도록, 실제 사용 가능한 다양성을 반영.
   - 초보자도 선택해서 바로 수정 가능한 구조 제공.
2. **브랜딩 품질 개선**
   - 랜딩/뷰어/템플릿의 톤·문구·시각 요소를 더 “전문적”으로 보이게 정리.
   - CTA, 신뢰 메시지, 핵심 가치 문구를 일관되게 통합.
3. **결과 중심 진행**
   - ‘한번에 끝내는’ 느낌이 아닌, 단계별 적용 후 피드백 가능한 구조 선호.
4. **원격지 확인 가능성 유지**
   - GitHub Pages/뷰어 기반으로 사용자가 어디서든 확인 가능해야 함.
5. **브랜딩 문서/템플릿 모두 통합 개선**
   - 단발성 변경보다 문서·페이지·템플릿이 같은 톤을 유지하도록 정렬 필요.

## 3) 현재까지 반영된 내용
- 템플릿 A/B/C 구조 개선(실사용형으로 재작성)
- 템플릿 다채로운 구성 가이드 추가
- Index + Viewer 페이지 UI/UX, CTA/탭/복사/상태/접근성 일부 개선
- GitHub 저장소는 `stomx` 기준으로 최종 반영 상태

## 4) 사용자가 원하는 다음 단계(요청)
- 앞으로는 **요구사항 정리 → 제안안 1~2개 제공 → 컨펌 요청** 순서로 진행
- 브랜딩/템플릿/문구를 한 번에 대충 고치는 방식보다,
  사용자가 선택 가능한 형태로 제시 후 확정.

## 5) 사용자 승인 요청 항목 (다음 작업 전)
아래 항목에 대해 사용자가 최종 확인해주길 요청함:

### A. 브랜딩 방향
- [ ] 톤앤드보이스 톤: `신뢰형/전문형/명확형`
- [ ] 시각 스타일: `딥 블루 + 신뢰형 포인트`
- [ ] 문구 규칙: `과장 금지, 일정/범위/조건 명시 우선`

### B. 템플릿 운영 방식
- [ ] Template A/B/C를 변형 가이드 기반으로 확장 적용
- [ ] 템플릿별로 초보자용/실무형 두 버전 동시 제공
- [ ] 문서/템플릿의 용어 통일(업로드용 용어, 문의 문구, FAQ 용어)

### C. 최종 산출물 포맷
- [ ] `index.html` + `viewer/index.html`는 전문형 구조 유지
- [ ] 템플릿 목록 페이지에서 바로 가이드 접근 가능
- [ ] 문구 정리본(`docs/*`)의 톤 통일 체크 반영

## 6) 다음 단계 제안(사용자 컨펌 후 실행)
- Step 1: 브랜딩 기준표(톤·색·버튼·문장 템플릿) 1장 고정
- Step 2: Template A/B/C에 대해 초보자/실무형 2버전씩 동시 생성
- Step 3: docs 핵심 문서 톤 통일 및 링크/문구 교차검증
